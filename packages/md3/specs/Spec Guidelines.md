# 문서 작성 기준
본 문서는 MD3 라이브러리 내 모든 스펙 문서의 작성 규칙과 제약 사항을 정의한다. 스펙 문서는 통일성 있고 정제된 형태를 유지해야 하며 정보 전달의 효율성을 최우선으로 고려한다.

# 문단 구조와 서식
문서는 시각적 서식에 의존하기보다 문단의 논리적 흐름을 통해 정보를 전달해야 한다. 마크다운 문법의 사용은 최소화하며 특히 텍스트 강조를 위한 굵은 글씨 사용을 전면 금지한다. 정보의 중요도는 기호가 아닌 문맥을 통해 자연스럽게 드러나게 작성한다. 단 구체적인 구현 예시나 API 인터페이스의 명확한 전달이 필수적인 경우에 한하여 코드 스니펫 블록의 사용을 예외적으로 허용한다.

# 제목 계층
문서 내의 제목은 단일 계층의 큰 제목만을 허용한다. 하위 계층의 소제목 사용은 엄격히 금지한다. 이는 각 단락이 독립적이고 명확한 주제를 다루도록 강제하여 문서 구조가 중첩되고 복잡해지는 것을 방지하기 위함이다.

# 어조와 문체
모든 문서는 대화형 어투나 설명조의 문장 종결을 피하고 객관적이고 권위 있는 문체를 사용한다. 구글 공식 문서와 같이 정제되고 전문적인 톤을 유지하며 불필요한 감정 표현이나 구어체 사용을 배제한다. 문장은 간결하고 명확하게 기술하여 기술적 사실만을 전달한다.
